Sessions vs Activations

LicenPro provides two complementary systems for tracking license usage: Sessions for real-time online tracking, and Activations for permanent device registration. Understanding when to use each is key to implementing the right licensing strategy.

Quick Comparison

Understanding the key differences between sessions and activations:

Sessions

Real-time online tracking - "Who is using the software RIGHT NOW?"

Activations

Permanent device registration - "Which devices are REGISTERED?"

Sessions Characteristics

Temporary

Disappears when app closes

Real-time

Tracks current online users

Heartbeat

Requires periodic check-ins

Concurrent Limits

"Max 5 users online"

Activations Characteristics

Permanent

Stays until manually deactivated

Device-bound

Tracks registered machines

One-time

Register once, use forever

Device Limits

"Max 3 devices registered"

Sessions Lifecycle

1

App Starts

User opens the application

When the user launches your application, it prepares to establish a session with the LicenPro server.

2

Connect

SessionManager.ConnectAsync()

The SDK connects to the server and creates a new session. The server tracks this user as "online".

3

Heartbeat

Automatic every 30 seconds

The SDK automatically sends heartbeats to keep the session alive. If heartbeats stop, the session times out.

4

Disconnect

Session removed

When the app closes or disconnects, the session is removed. The user slot becomes available for others.

Activations Lifecycle

1

First Run

New device detected

When the application runs on a new device for the first time, it detects that this device hasn't been registered.

2

Activate

ActivationManager.ActivateDeviceAsync()

The SDK sends the device fingerprint to the server. If activation slots are available, the device is registered.

3

Registered

Device stored permanently

The device is now permanently registered. It can use the software even offline (with license file).

4

Deactivate

Admin removes manually

To free up an activation slot, an admin must manually deactivate the device from the dashboard.

SDK Integration

C# - Sessions (Real-time tracking)
// Connect when app starts
var result = await SessionManager.ConnectAsync(licenseKey, Environment.UserName);

if (result.Success)
{{ '{' }}
    Console.WriteLine($"Connected! Session: {{ '{' }}result.SessionId{{ '}' }}");
    // Heartbeats are sent automatically
{{ '}' }}

// Disconnect when app closes
await SessionManager.DisconnectAsync();
C# - Activations (Device registration)
// Activate device (call once, on first run)
var success = await ActivationManager.ActivateDeviceAsync(licenseKey);

if (success)
{{ '{' }}
    Console.WriteLine("Device registered successfully!");
    // Device is now permanently registered
{{ '}' }}
else
{{ '{' }}
    Console.WriteLine("Activation failed - device limit may be reached");
{{ '}' }}
C# - Both Systems (Recommended for enterprise)
public async Task StartApplicationAsync(string licenseKey)
{{ '{' }}
    // Step 1: Register device (first time only)
    var activationResult = await ActivationManager.ActivateDeviceAsync(licenseKey);
    
    if (!activationResult)
    {{ '{' }}
        // Device limit reached
        MessageBox.Show("Maximum devices reached. Deactivate another device first.");
        return;
    {{ '}' }}
    
    // Step 2: Connect session (every time app runs)
    var sessionResult = await SessionManager.ConnectAsync(licenseKey, Environment.UserName);
    
    if (!sessionResult.Success)
    {{ '{' }}
        // Too many concurrent users
        MessageBox.Show("Too many users online. Try again later.");
        return;
    {{ '}' }}
    
    // App is ready!
    Console.WriteLine("Device registered + Session connected");
{{ '}' }}
Best Practice: Use Activations to limit how many devices can use a license, and Sessions to limit how many can be online simultaneously. This gives you maximum control.
