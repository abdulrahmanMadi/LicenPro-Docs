WinForms Integration Guide

Step-by-step guide to adding LicenPro licensing to your Windows Forms applications. Learn how to validate licenses at startup, handle activation, and provide a seamless user experience.

Prerequisites

✓
Visual Studio 2019 or later

With .NET desktop development workload installed

✓
.NET 6+ or .NET Framework 4.7.2+

WinForms project targeting a supported framework

✓
LicenPro Product Created

With RSA keys generated and public key downloaded

Step 1: Install NuGet Package

Add the LicenPro SDK to your WinForms project:

Package Manager Console
Install-Package LicenPro

Step 2: Modify Program.cs

Update your Program.cs to validate the license before showing the main form:

1
Change Main to async Task

Change "static void Main()" to "static async Task Main()" to support async license operations.

2
Initialize SDK

Create SdkSettings with ServerBaseEndpoint set to your API base URL (for example https://licenpro.runasp.net/api), then call SdkConfiguration.Initialize(settings).

3
Load Public Key

Create LicensingSecrets with your public key and call LicenseUtils.LoadLicensingSecrets(secrets).

4
Check for License File

Look for "license.bin" in the application directory. If found, load and validate it using LicenseManager.LoadLicenseAsync(licensePath).

5
Show Appropriate Form

If license is valid, show MainForm. Otherwise, show LicenseActivationForm.

Step 3: Create License Activation Form

Create a form for users to enter and activate their license:

License Key TextBox

Input field for the license key

Activate Button

Triggers validation and activation

Status Label

Shows validation progress and results

In the Activate button click handler:

1
Validate Online

Use LicenseClient.ValidateAsync() (recommended) or load a license.bin file using LicenseManager.LoadLicenseAsync(filePath).

2
Download License File

If validation succeeds, store the license file in a stable location (for example C:\ProgramData\YourApp\license.bin). In most products, the license file is distributed to the customer from the dashboard (downloaded by an admin and sent to the user).

3
Open Main Form

Hide activation form and show the main application form

Best Practices

Async/Await: Always use async methods for license operations to keep the UI responsive. Never block the UI thread with synchronous calls.
Error Handling: Wrap license operations in try-catch blocks and provide user-friendly error messages.
Security: Don't store the license key in plain text. Use the license file (.bin) which is cryptographically signed and harder to tamper with.

Secure Credential Caching (Advanced)

For a seamless user experience, you can securely cache the License Key and Public Key after the first successful validation. On subsequent launches, the app can auto-validate without prompting the user.

1
After First Validation

When validation succeeds, encrypt and store the LicenseKey and PublicKey using Windows DPAPI (ProtectedData.Protect) with DataProtectionScope.CurrentUser. This ensures credentials can only be decrypted by the same Windows user on the same machine.

2
Store Protected License

Also cache the license file in encrypted form (e.g., license.secure.bin) using LicenseFileProtector.ProtectAsync. This creates a tamper-resistant local copy tied to the license key.

3
Auto-Validate on Startup

On app launch, check for the cached credentials and protected license. If present, load them automatically and call ValidateOfflineAsync (fast, no network). The app starts instantly without user interaction.

4
Security Benefits

The cached data is useless if copied to another machine or user account. Even if the files are extracted, they cannot be decrypted without the original Windows user context.

UX Tip: This pattern provides both security and convenience—users validate once, then enjoy instant startup times on subsequent launches. The test app in the LicenPro SDK repository demonstrates this implementation.
