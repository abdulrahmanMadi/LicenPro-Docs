RSA Key Configuration

Learn how RSA cryptographic keys work in LicenPro to secure your licenses. RSA keys ensure that licenses cannot be forged or tampered with, providing strong protection for your software.

What are RSA Keys?

RSA (Rivest-Shamir-Adleman) is an asymmetric cryptographic algorithm that uses a pair of keys: a public key and a private key. In LicenPro, these keys are used to sign and verify licenses.

Private Key

Used by the server to sign licenses. Keep this secret and secure - never share it!

Public Key

Embedded in your application to verify license signatures. Safe to distribute.

How License Signing Works

Here's the flow of how RSA keys protect your licenses:

1

License Creation

When you create a license, the server generates license data (user info, expiration, features, etc.)

2

Digital Signature

The server uses your private key to create a digital signature of the license data

3

License Distribution

The signed license (key or file) is delivered to your customer

4

Verification

Your application uses the public key to verify the signature is valid

Generating RSA Keys

Follow these steps to generate RSA keys for your product:

1
Go to Product Settings

Navigate to your product and click on the "Settings" tab.

2
Find RSA Keys Section

Scroll down to the "RSA Key Configuration" section.

3
Generate New Keys

Click "Generate New Key Pair". LicenPro will create a 2048-bit RSA key pair.

4
Download Public Key

Click "Download Public Key" to get the .pem file. You'll embed this in your application.

5
Backup Private Key (Optional)

For disaster recovery, you can download the private key. Store it in a secure location.

Security Warning: Never share your private key. Never embed it in your application. Never commit it to source control.

Using the Public Key

To use the public key in your application:

1
Download the .pem file

Get the public key from Product Settings → Download Public Key

2
Extract the Base64 content

Open the .pem file and copy the Base64 string between the header and footer lines

3
Create LicensingSecrets

Create a LicensingSecrets object with the PublicKey property set to your Base64 key

4
Load the secrets

Call LicenseUtils.LoadLicensingSecrets(secrets) to initialize

Embedding vs File: For desktop apps, embedding the public key as a string is more secure. For web apps, use a file.

Security Best Practices

Use Strong Key Sizes

LicenPro uses 2048-bit RSA keys by default, which provides strong security.

Secure Key Storage

Store private key backups in encrypted storage like a password manager or secure vault.

Obfuscate Public Key

Consider obfuscating the public key in your app to make reverse engineering harder.

Rotate Keys Periodically

Rotate keys annually or immediately if you suspect a compromise.
