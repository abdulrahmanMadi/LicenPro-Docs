RSA Key Management

LicenPro uses asymmetric cryptography (RSA) to ensure that only you can issue valid licenses for your software.

Private Key

Stored securely on our servers. Used to "sign" digital license files.

Public Key

Hardcoded in your application. Used to verify the server's signature.

Security Principles

Asymmetric Signing

Because the Public Key can only verify (not create) a signature, your app can safely check license validity even when disconnected from the internet without exposing your master signing key.

Rotation & Backup

You can regenerate your RSA keys at any time from the Product Settings. However, please be aware:

Breaking Changes

Rotating keys will invalidate all existing licenses signed with the old key. You must update your application's source code with the new Public Key before re-issuing licenses to users.

// Example: Embedding Public Key in C#
var publicKey = "MIIBIjANBgkqhkiG9w0BAQEFAAOCAQ8AM...";
LicenseUtils.LoadPublicKey(publicKey);
