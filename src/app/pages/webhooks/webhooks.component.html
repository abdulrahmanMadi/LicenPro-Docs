Webhooks

Webhooks allow you to receive real-time notifications when events occur in your LicenPro account. Use them to automate workflows, sync with external systems, or trigger custom actions.

Available Events

Webhooks are planned for a future release. In the current backend build, there are no webhook delivery endpoints/controllers enabled yet.

Workaround today: if you need integrations (CRM/billing/telemetry), poll the REST API endpoints (licenses/status, sessions, activations) or write directly from your own dashboard UI when actions occur.

Webhook Payload

Webhook payload formats will be documented here once webhook delivery is enabled. For now, use the REST API as the integration surface.

Setting Up Webhooks

This section will be enabled when webhooks are shipped.

1
Create an Endpoint

Set up an HTTPS endpoint on your server to receive webhook events. The endpoint must be publicly accessible.

2
Configure in Dashboard

Go to Product Settings → Webhooks and add your endpoint URL.

3
Select Events

Choose which events you want to receive notifications for.

4
Set Secret Key

Configure a secret key to verify webhook signatures and ensure requests are from LicenPro.

5
Test Your Endpoint

Use the "Send Test" button to verify your endpoint is receiving and processing webhooks correctly.

Verifying Webhook Signatures

Signature verification will be documented when webhooks are shipped.

Security: Always verify webhook signatures before processing. This prevents attackers from sending fake webhook events to your endpoint.

Retry Policy

Retry policy will be documented when webhooks are shipped.

1
First Retry

After 1 minute

2
Second Retry

After 5 minutes

3
Third Retry

After 30 minutes

4
Final Retry

After 2 hours (then marked as failed)

Respond Quickly: Return a 200 OK response within 30 seconds. Process webhook data asynchronously if needed to avoid timeouts.
