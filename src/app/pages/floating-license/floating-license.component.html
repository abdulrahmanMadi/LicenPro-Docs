Floating License

Floating licenses (also called concurrent licenses) allow a set number of users to use your software simultaneously. Perfect for organizations where not everyone needs access at the same time.

What is a Floating License?

A Floating License allows multiple users to share a pool of license seats. For example, a 10-seat floating license means up to 10 users can use the software at the same time, but it can be installed on unlimited machines. When a user closes the application, their seat becomes available for another user.

Concurrent Users

Multiple users share a pool of available seats

Cost Effective

Organizations pay for concurrent usage, not total users

Unlimited Installs

Install on any number of machines

Dynamic Allocation

Seats are released when users close the app

How Floating Licenses Work

1

User Launches App

Application requests a seat from the license server

2

Seat Check

Server checks if seats are available in the pool

3

Seat Allocation

If available, a seat is allocated to the user

4

Heartbeat

App sends periodic heartbeats to maintain the seat

5

Seat Release

When app closes (or heartbeat stops), seat returns to pool

Creating a Floating License

1
Navigate to Licenses → Create License

Go to the license creation page.

2
Select "Floating" License Type

Choose Floating from the license type dropdown.

3
Set Number of Seats

Configure how many concurrent users can use the software (e.g., 5, 10, 25, 100 seats).

4
Configure Heartbeat Interval

Set how often the app should check in (recommended: 1-5 minutes). Shorter intervals = faster seat release when app crashes.

5
Enter Organization Details

Add the organization name and admin contact for the floating license.

SDK Integration

C# - Floating License Implementation
using LicenPro;
using LicenPro.Models.License;

// Initialize with floating license support
var settings = new SdkSettings
{{ '{' }}
    ServerBaseEndpoint = "https://YOUR_DOMAIN/api",
    HeartbeatIntervalMinutes = 2  // Check in every 2 minutes
{{ '}' }};
SdkConfiguration.Initialize(settings);

// Request a seat
var result = await LicenseManager.CheckoutSeatAsync(licenseKey);

if (result.Status == SeatStatus.Acquired)
{{ '{' }}
    Console.WriteLine("Seat acquired successfully!");
    // Start your application
    
    // The SDK automatically sends heartbeats
    // When app closes, call:
    await LicenseManager.ReleaseSeatAsync();
{{ '}' }}
else if (result.Status == SeatStatus.NoSeatsAvailable)
{{ '{' }}
    Console.WriteLine("All seats are in use. Please try again later.");
    ShowNoSeatsDialog();
{{ '}' }}
Important: Always call ReleaseSeatAsync() when your application closes to immediately free the seat. If not called, the seat will be released after the heartbeat timeout.

Best Practices

Seat Estimation: Analyze your customer's usage patterns. Typically, organizations need seats for 20-30% of their total users if not everyone uses the software simultaneously.
Graceful Handling: When no seats are available, show a friendly message and optionally allow users to queue or be notified when a seat becomes available.
