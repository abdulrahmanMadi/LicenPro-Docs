How it Works

Unlike Perpetual licenses which bind to a specific user or machine, a Floating license represents a "Seat." When User A launches the app, they take 1 seat. When they close it, the seat returns to the pool for User B.

Session Heartbeat

Floating licenses require an active internet connection. The SDK sends a "heartbeat" every few minutes to maintain the seat. If the server loses connection for more than 10 minutes, the seat is automatically made available again.

// Initialize as Floating
var options = new LicenseClientOptions {{ '{' }}
    ExpectedType = LicenseType.Floating,
    SessionHeartbeatInterval = 5 // minutes
{{ '}' }};
