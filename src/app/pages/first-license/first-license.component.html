Generate your first license

A license binds entitlements to a customer: type, seats, expiry, software release, and optional feature sets. After creation you distribute the license key and signed license.bin for SDK offline validation. Complete first product and RSA keys before issuing licenses.

Estimated time: 10–15 minutes

Includes choosing license model, release line, and downloading artifacts from license details.

Prerequisites

Product with at least one software release. RSA keys generated for that product (unsigned licenses cannot produce a valid license.bin).

What you get after issuance

License key

Human-readable key (e.g. LP-…) for activation UI and online flows.

license.bin

RSA-signed payload file for offline SDK validation.

Public key

From product settings—embed in your app with the SDK.

Open Create License

From the dashboard: Products → select your product → Licenses → Create License, or use the global Licenses list and pick the product context. The form is a two-step wizard: Basic Info and Details.

1

Step 1 — Basic Info

License type (required)—choose the model that matches how you sell access:

  • Perpetual — multi-user, indefinite use; online or offline mode.
  • Concurrent — shared team key; max simultaneous users via activations.
  • Node-locked — bound to one machine (hardware captured on first run).
  • Subscription — recurring term; duration in days or hours.
  • Floating — enterprise seat pool enforced by live sessions.
  • Trial — time-limited evaluation with trial period fields.

Model-specific guides: sidebar License models or perpetual, trial, subscription, floating, concurrent, node-locked.

Product (required if not already in product context)—select which application this license belongs to.

Software release (required)—pick the release line created with your product (e.g. 1.0.0 Stable). Validation and updates use this id.

Issuer (optional)—company or vendor name printed on license metadata.

Issued to — depends on type: email for many models, end-user picker for node-locked, auto for some perpetual modes. Must match what the SDK expects during validation when identity is enforced.

Perpetual mode (perpetual only):

  • Online — sessions and activations tracked against the API.
  • Offline — file-based; note dashboard restrictions for offline perpetual creation on some product access types.
2

Step 2 — Details

  • License name (required) — internal label for support and search.
  • Expiry / duration — model-specific (subscription length, trial period, or none for perpetual).
  • Seats & limits — max activations, max active users (floating), hardware binding options where shown.
  • Entitlement set (optional) — package of features configured under the product.
  • Notes (optional) — internal rich-text notes for operators.

Click Save License (or Create License). On success the UI shows the generated license key with a copy action.

3

Download license.bin

Open the license from the list or success screen → Download license file (or equivalent on license details). The file is named {license-name}_license.bin. Ship it with your installer or import path in the SDK via LicenseClientOptions.LicenseFilePath.

There is no .lic extension in the default download—use the license.bin artifact from the dashboard.

Product access type and license types

Opened products may restrict certain perpetual online types in the create form. Associated products tie licenses to organization context. If the UI blocks a type you need, review product access on the product settings or convert access type before issuing.

Distribute to customers

  • Online-first apps — license key plus SDK calls to POST /api/Licenses/validate with product X-API-KEY.
  • Offline-capable apps — license.bin, license key, and embedded public key; SDK verifies RSA signature locally.
  • Never ship the RSA private key or your operator JWT in customer binaries.
License key format

Keys are human-readable (commonly LP-… segments) for copy-paste activation flows.

REST API

Automate issuance with operator JWT routes under /api/Licenses/.... See Licenses API and Licenses (vendor).
