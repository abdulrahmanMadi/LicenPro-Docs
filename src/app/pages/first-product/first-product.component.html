Create your first product

A product is the application you license in LicenPro. It anchors your first software release, RSA signing keys, product API credentials, entitlements, and every license you issue. This guide matches the Create Product modal in the dashboard.

Estimated time: 5–10 minutes

Requires an organization and dashboard access. RSA keys are generated after the product exists—see RSA keys.

Before you start

  • Sign in at app.licenpro.tech and complete your first organization if needed.
  • Open Products in the sidebar (/dashboard/products) and click Create Product.
  • Product limits depend on your plan; upgrade if the create action is blocked.

Create Product form

The modal captures product identity, the initial release (required), access rules, and optional branding.

1

Product name

Product name (required, max 100 characters)—unique label in your account (e.g. My Awesome Software).

2

Version setup

Initial version (required) creates your first software release:

  • Version format — Semantic (e.g. 1.0.0), pre-release, simple, or build-based.
  • Release level — Stable, Beta, Alpha, or RC.

Every license and update check references a softwareReleaseId from this line.

3

Product access type

  • Associated (Organization) — licenses are tied to specific organizations; only members of assigned orgs use the product.
  • Opened (All users) — any holder of a valid license key can use the product across organizations.

Access type affects which license models are available when you issue keys.

4

Description & image (optional)

Rich-text description (max 1000 characters) and optional product image (PNG, JPG, GIF, WebP, max 2MB).

5

Create Product

Click Create Product. You enter the product workspace: overview, features, entitlements, releases, licenses, users, access matrix, and audit logs.

After creation

  1. Generate RSA keys — Product → Settings → License signing keys (RSA). Mandatory before downloading signed license.bin. See RSA keys.
  2. Product API key — copy the product X-API-KEY for anonymous validation routes (distinct from account API keys in Settings).
  3. Issue a license — Generate your first license.
