.NET SDK Integration Guide

Complete guide to integrating LicenPro licensing into your .NET applications. The SDK supports .NET 6+, .NET Framework 4.7.2+, and works with Console, WinForms, WPF, and ASP.NET applications.

Installation

Install the LicenPro SDK via NuGet Package Manager:

Package Manager Console
Install-Package LicenPro
.NET CLI
dotnet add package LicenPro

Required Dependencies

The LicenPro SDK requires the following NuGet packages for configuration management:

.NET CLI
dotnet add package Microsoft.Extensions.Configuration --version 8.0.0
dotnet add package Microsoft.Extensions.Configuration.Abstractions --version 8.0.0
dotnet add package Microsoft.Extensions.Configuration.Json --version 8.0.0
dotnet add package Microsoft.Extensions.Configuration.Binder --version 8.0.0
Important: Without these packages, you will get the error: Could not load file or assembly 'Microsoft.Extensions.Configuration'

Quick Start Steps

Follow these steps to integrate licensing into your application:

1

Initialize SDK Settings

Configure the server endpoint

Create an SdkSettings object with your server endpoint (https://licenpro.runasp.net/api) and call SdkConfiguration.Initialize(settings).

2

Load Your Public Key

Configure license validation

Download your public key from Product Settings in the dashboard. Create a LicensingSecrets object with the public key and call LicenseUtils.LoadLicensingSecrets(secrets).

3

Validate License

Check license validity

Use LicenseManager.ValidateLicenseAsync(licenseKey) for online validation, or LicenseManager.LoadLicenseAsync(filePath) for offline validation with a license file.

4

Check License Status

Handle the validation result

The result contains a Status property (Valid, Expired, Revoked, Invalid, etc.) and a License object with details like UserName, ExpirationDate, and Features.

Key Classes and Methods

SdkConfiguration

Initialize SDK with server endpoint and settings

LicenseManager

Validate, load, and manage licenses

LicenseUtils

Load public keys and licensing secrets

BaseLicense

Base class for all license types

License Types

Cast the BaseLicense to specific types for additional properties:

StandardLicense

Single-user perpetual license with UserName property

TrialLicense

Time-limited evaluation license

SubscriptionLicense

Recurring billing with Tier property

FloatingLicense

Concurrent user seats with MaxSeats property

Error Handling

Handle different license statuses appropriately:

Valid

License is valid - proceed with application

Expired

Show renewal dialog to user

Revoked

License was revoked - exit application

NetworkError

Try offline validation with license file

Best Practices

Use Async Methods: Always use async/await for license operations to keep your UI responsive.
Cache License Files: Download and store license files locally for offline validation support.
Protect Your Public Key: While safe to distribute, consider obfuscating the public key in your application to make reverse engineering harder.
