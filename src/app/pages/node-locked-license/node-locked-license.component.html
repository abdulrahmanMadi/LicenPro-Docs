Node-locked license

A node-locked license binds entitlement to one machine via hardware fingerprint. Copying the license file to another PC fails validation—ideal for per-install desktop software and regulated environments.

Business model

Node-locked is the standard “one license = one computer” SKU—common in vertical software, medical devices, and industrial tooling.

  • Per-install retail — customer buys one key per workstation.
  • OEM / appliance — license pre-bound to shipped hardware.
  • Compliance — audit trail that software runs only on certified machines.
  • Anti-sharing — stronger than open perpetual keys for casual piracy.

Benefits

  • For vendors — limits casual key sharing; Device tab shows exact binding for support.
  • For customers — clear “this PC only” entitlement; transfer workflow for hardware upgrades.
  • For support — pending-bind queue approves first-run binding; block compromised devices.

Logic of use

At issue — hardware id may be set in wizard or captured on first activation (auto-bind). Server and license.bin both carry binding material.

Online validation — server compares client fingerprint; mismatch returns hardware error. Offline validation — signature + embedded hardware in file.

One activation per license typical—Activations tab shows single primary device; Device tab is the operator focus.

Example lifecycle

Scenario: First-run auto-bind

Customer receives key + file. First launch captures HWID, sends bind request. Operator approves in pending-bind queue (or auto-approves). License locked to that PC. Copying file to second PC fails validation.

Scenario: Hardware upgrade

User replaces motherboard. Support opens Device tab → unbind/transfer → user re-activates on new machine. Same commercial license, new fingerprint.

Lifecycle stages

  1. Issue — type Node-locked; pick end user / release context.
  2. Bind — at create or first run (pending-bind workflow).
  3. Active — app runs only on bound hardware.
  4. Transfer — operator unbind for legitimate moves.
  5. Block device — security response on Activations/Device.
  6. Revoke — end contract from Overview.

Dashboard workflow

  • Create → Node-locked → issued-to picker from release.
  • Detail → Device tab (binding, transfer, history).
  • Activations — device rows; block suspicious entries.
  • Sessions — when online validation enabled alongside binding.
