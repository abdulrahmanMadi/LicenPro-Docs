Node-Locked License

A node-locked license is bound to a single device using a hardware fingerprint (Hardware ID). This is the strictest offline licensing model and is ideal for preventing key sharing.

How Node-Locked Licensing Works

Node-locked licensing ties a license to a device fingerprint. During validation/activation, the SDK sends (or derives) a HardwareId. The backend enforces that the license can only be used on that hardware.

Strong binding

Once bound, the license can only validate on the same hardware fingerprint.

Offline-friendly

After you distribute license.bin, validation can be performed offline using the product public key.

Remote control (optional)

When online validation is enabled, devices can be blocked and licenses can be revoked from the dashboard.

Create a Node-Locked License (Dashboard)

1
Collect the Hardware ID

In your desktop app, display the hardware fingerprint to the user (or capture it automatically). This value is sent as HardwareId during validation/activation.

2
Create License

Go to Licenses → Create License, set Type = NodeLocked, and set Issued To (customer identity).

3
Bind hardware (recommended)

Enter the device HardwareId into the license when issuing. This avoids “first-use auto-binding” to an unexpected device.

4
Download license.bin

Assign the license to a release, then download the signed license file (.bin) and send it to the customer.

Security note: If a node-locked license has no hardware bound yet, the backend can auto-bind it on first validation/activation. For strict security, bind the hardware at issuance time.

SDK Integration (Recommended)

Use LicenseClient to perform offline validation, optional online validation, and activation.

C# - Validate + activate (NodeLocked)
using LicenPro.Enums;
using LicenPro.SDK;

var client = new LicenseClient(new LicenseClientOptions
{{ '{' }}
    LicenseFilePath = @"C:\ProgramData\YourApp\license.bin",
    PublicKey = "YOUR_PRODUCT_PUBLIC_KEY",
    LicenseKey = "XXXXX-XXXXX-XXXXX-XXXXX-XXXXX",
    ExpectedLicenseType = LicenseType.NodeLocked,

    // Optional but recommended for node-locked
    ForceOnlineValidation = true,
    ServerEndpoint = "https://YOUR_DOMAIN/api"
{{ '}' }});

var result = await client.ValidateAsync();
if (!result.IsValid)
    throw new Exception(result.ErrorMessage);

// If activation is required/available, this calls POST /api/activations
await client.TryActivateDeviceAsync();

Troubleshooting

HardwareMismatch

Cause: The license is bound to a different HardwareId.

Fix: Issue a new node-locked license for the correct hardware, or reset/rebind the license from the dashboard if your business rules allow it.

DeviceBlocked

Cause: The device has been blocked for this license.

Fix: Unblock the device in the dashboard, then revalidate online.
