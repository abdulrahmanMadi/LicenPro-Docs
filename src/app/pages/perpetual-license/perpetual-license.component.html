Perpetual License

A multi-user license model that never expires. Ideal for desktop applications where customers pay once and use the software forever.

Never Expires

Permanent access once issued.

Multi-User

Configurable seat/activation limits.

Offline First

No internet required for validation.

Overview

The Perpetual License model grants a customer permanent rights to use a specific version of your software. It is the gold standard for professional tools and "buy-it-once" software products.

Hybrid Validation Strategy

While perpetual licenses work offline via RSA signatures, you can enable Online Validation for heartbeat monitoring, usage tracking, and remote revocation.

How it works

1

Create License

In the dashboard, select Perpetual. Fill in the customer email and optional expiration (e.g., for maintenance coverage).

2

Download Payload

Download the license.bin file. This encrypted payload contains the signed license data.

3

SDK Validation

Your app validates the payload locally using your product's Public Key. No backend request is required unless you force online mode.

SDK Integration

// Simple Perpetual Validation
var client = new LicenseClient(new LicenseClientOptions {{ '{' }}
    LicenseFilePath = "license.bin",
    PublicKey = "MIIB...",
    ExpectedType = LicenseType.Perpetual
{{ '}' }});

var result = await client.ValidateAsync();
Signature Security

The LicenPro SDK uses RSA-SHA256 signatures. If a user tries to modify the license file (e.g., changing the expiration date), the signature verification will fail immediately.
