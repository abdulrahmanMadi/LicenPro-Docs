Perpetual License

A Perpetual License is a single-user license that never expires. It's the most common licensing model for desktop applications - customers pay once and use the software forever. Perfect for traditional software sales with no recurring fees.

What is a Perpetual License?

A Perpetual License grants a single user permanent access to your software. Unlike subscriptions or trials, it doesn't expire and works completely offline by default. Each license is tied to a specific user via their email address.

Never Expires

License remains valid forever unless you set an optional expiration date

Single User

One license per customer, tied to their email address

Works Offline

No internet required - validates locally using RSA signatures

Optional Online Mode

Enable heartbeat monitoring for usage tracking and analytics

When to Use Perpetual Licenses

Perpetual licenses are ideal for:

Desktop Applications

Traditional desktop software where customers expect to pay once and own forever.

Professional Tools

Design software, development tools, or professional applications with one-time purchase model.

Offline Environments

Software used in secure or offline environments where internet isn't available.

Simple Sales Model

When you want a straightforward pricing model without subscription management.

Creating a Perpetual License

Follow these steps to create a perpetual license:

1
Navigate to Licenses

Go to Licenses → Create License from your dashboard.

2
Select License Type

Choose "Perpetual" from the license type dropdown.

3
Enter User Information

Add the customer's name and email. The email is used as the stable identifier for validation.

4
Set Expiration (Optional)

Leave blank for perpetual validity, or set a date for limited-time perpetual licenses (e.g., for major version upgrades).

5
Configure Features (Optional)

Assign specific features to enable or disable functionality based on the license tier.

6
Generate and Deliver

Create the license and download the .bin file. Send it to your customer via email or download link.

// Create the license client

// Validate the license (offline by default)
var result = await client.ValidateAsync();

if (result.IsValid)

Offline vs Online Mode: By default, perpetual licenses work offline. Set ForceOnlineValidation = true to enable session tracking, heartbeat monitoring, and real-time usage analytics in your dashboard.

Best Practices

User Email as Identifier: Use the customer's email as the UserName field when creating the license. Your app should validate using the same email string to ensure license belongs to the right user.
Graceful Expiration Handling: Even perpetual licenses can have optional expiration dates. Always check License.ExpirationDate and handle expired licenses gracefully with renewal prompts.
Feature-Based Tiers: Use features to create license tiers (Basic, Pro, Enterprise). This lets you sell different price points with the same codebase.
